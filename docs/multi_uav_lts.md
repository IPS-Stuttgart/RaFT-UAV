# Multi-UAV LTS workflow

This note records the repository-side support for the Beyond Strong Baseline:
Multi-UAV Tracking Competition (LTS). The official leaderboard is on Codabench,
so local validation here only checks packaging and format readiness; HOTA, MOTA,
and IDF1 require an official upload.

## RaFT-UAV utilities

Inventory data or template archives:

```bash
PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python -m raft_uav.multi_uav_lts.cli inventory \
  /mnt/lexar4tb/multi_uav_lts/downloads/Test.zip \
  --output-json outputs/multi_uav_lts_test_inventory.json
```

Validate a Codabench submission ZIP:

```bash
PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python -m raft_uav.multi_uav_lts.cli validate-submission \
  submission.zip \
  --template-zip /mnt/lexar4tb/multi_uav_lts/downloads/submission.zip \
  --output-json submission_validation.json \
  --file-summary-csv submission_file_summary.csv
```

Package root-level `.txt` predictions into a sorted, normalized submission:

```bash
PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python -m raft_uav.multi_uav_lts.cli package-submission \
  predictions \
  --template-zip /mnt/lexar4tb/multi_uav_lts/downloads/submission.zip \
  --output-zip submission.zip \
  --normalize \
  --sort-rows \
  --output-json submission_validation.json \
  --file-summary-csv submission_file_summary.csv
```

Audit prediction coverage before packaging. This catches missing, extra, empty,
malformed, unsorted, and out-of-range-frame prediction files before the packager
fills missing submission-template entries with empty files. Passing
`--sequence-root` lets the audit compare prediction frame IDs against actual
image counts for each sequence:

```bash
PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
raft-uav-multi-uav-lts-coverage-audit \
  /mnt/lexar4tb/multi_uav_lts/outputs/official_baseline_via_first_init/predictions \
  --template-zip /mnt/lexar4tb/multi_uav_lts/downloads/submission.zip \
  --sequence-root /mnt/lexar4tb/multi_uav_lts/extracted/TestImages \
  --output-json prediction_coverage_audit.json \
  --row-csv prediction_coverage_rows.csv \
  --require-ready
```

Extract train first-frame labels for tracker initialization:

```bash
PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python -m raft_uav.multi_uav_lts.cli first-frame-labels \
  --truth-dir /mnt/lexar4tb/multi_uav_lts/extracted/TrainLabels \
  --output-dir /mnt/lexar4tb/multi_uav_lts/extracted/TrainLabels_FirstFrameOnly \
  --output-json /mnt/lexar4tb/multi_uav_lts/outputs/train_first_frame_labels_summary.json
```

Score a train-split prediction directory or ZIP with the local MOTA-like
diagnostic evaluator:

```bash
PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python -m raft_uav.multi_uav_lts.cli score-predictions \
  /mnt/lexar4tb/multi_uav_lts/outputs/train_smoke_official_baseline_BB1/predictions \
  --truth-dir /mnt/lexar4tb/multi_uav_lts/extracted/TrainLabels \
  --sequences BB1_00 BB1_01 \
  --output-json /mnt/lexar4tb/multi_uav_lts/outputs/train_smoke_official_baseline_BB1/score.json \
  --sequence-summary-csv /mnt/lexar4tb/multi_uav_lts/outputs/train_smoke_official_baseline_BB1/score_by_sequence.csv
```

The local scorecard is a train-split tuning aid, not the official Codabench
HOTA/MOTA/IDF1 evaluator.

## Upstream official-baseline runner

The strong local submission was generated by the upstream
`YOLOv12-BoT-SORT-ReID` baseline and packaged by RaFT-UAV:

```bash
source /mnt/lexar4tb/multi_uav_lts/venvs/yolov12-botsort-py312/bin/activate

PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python scripts/run_multi_uav_lts_official_baseline.py \
  --work-root /mnt/lexar4tb/multi_uav_lts \
  --python python \
  --device 0 \
  --img-size 1920 \
  --normalize \
  --sort-rows
```

To run in shards:

```bash
SHARD_COUNT=2 SHARD_INDEX=0 GPU_ID=0 python scripts/run_multi_uav_lts_official_baseline.py
SHARD_COUNT=2 SHARD_INDEX=1 GPU_ID=1 python scripts/run_multi_uav_lts_official_baseline.py
```

After all shards finish, package without rerunning inference:

```bash
PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python scripts/run_multi_uav_lts_official_baseline.py \
  --work-root /mnt/lexar4tb/multi_uav_lts \
  --package-only \
  --normalize \
  --sort-rows
```

For train-split smoke tests, omit the test template and restrict the sequence
set:

```bash
source /mnt/lexar4tb/multi_uav_lts/venvs/yolov12-botsort-py312/bin/activate

PYTHONPATH=src RAFT_UAV_SKIP_RUNTIME_HOOKS=1 \
python scripts/run_multi_uav_lts_official_baseline.py \
  --work-root /mnt/lexar4tb/multi_uav_lts \
  --sequence-root /mnt/lexar4tb/multi_uav_lts/extracted/TrainImages \
  --first-frame-label-dir /mnt/lexar4tb/multi_uav_lts/extracted/TrainLabels_FirstFrameOnly \
  --output-dir /mnt/lexar4tb/multi_uav_lts/outputs/train_smoke_official_baseline_BB1 \
  --predictions-dir /mnt/lexar4tb/multi_uav_lts/outputs/train_smoke_official_baseline_BB1/predictions \
  --no-template \
  --sequences BB1_00 BB1_01 \
  --python python \
  --device 0 \
  --img-size 1920 \
  --normalize \
  --sort-rows
```
